Codexラッパー基盤
# 要件定義

## 0. 文書情報
- プロジェクト名: `codex-cli-wrapper-base`
- 文書版: `v0.36`
- 更新日: `2026-02-14`

## 1. 目的
Codex CLI を対話モードで扱うための最小ラッパーを提供する。第一段階では、メインウィンドウ単体で次を成立させる。
- Codex の出力がメインウィンドウに表示されること
- メインウィンドウから Codex CLI にコマンド送信できること

## 2. スコープ
### 2.1 変更対象（in-scope）
- Python GUI 起動（メインウィンドウ）
- `pywinpty` による対話TTYセッション開始
- `codex --no-alt-screen` の起動
- 出力表示（メインウィンドウ）
- メインウィンドウからのコマンド送信
- セッションログ保存（`stdout.log` `commands.log` `meta.json`）

### 2.2 非対象（non-goals）
- 複数Codexセッションの同時管理
- 別窓ミラー表示
- 音声入力
- 配布用exe化
- 自動要約機能

## 3. 前提条件
- OS: Windows
- `codex` コマンドが利用可能
- Python 3.10 以上
- `pywinpty` がインストール済み

## 4. 機能要件（Delta Requirements）
### REQ-S1-001 GUI起動
- 変更理由: 対話制御のUI基盤を作るため
- 受け入れ条件:
  - `python codex_cli_wrapper_gui.py` でウィンドウが起動する
  - ステータスに `codex --version` の確認結果が表示される

### REQ-S1-002 対話TTY起動
- 変更理由: `stdin is not a terminal` を回避して対話モードを成立させるため
- 受け入れ条件:
  - `Start Codex` でTTYセッションが開始される
  - セッション起動後に `codex --no-alt-screen` が送信される

### REQ-S1-003 出力表示
- 変更理由: メインウィンドウ上で対話状況を確認できるようにするため
- 受け入れ条件:
  - Codex出力がメインウィンドウに表示される
  - フィルターで出力行を除外せず、受信テキストをそのまま表示する

### REQ-S1-004 コマンド送信
- 変更理由: メインウィンドウから対話を継続できるようにするため
- 受け入れ条件:
  - メインウィンドウの入力欄から送信したコマンドがTTYへ届く
  - `Send` ボタン1回押下で送信される（2回押下を不要とする）
  - 同一入力をラッパー側で重複表示しない（Codex側表示のみを採用）

### REQ-S1-005 セッション記録
- 変更理由: 検証と再現確認を可能にするため
- 受け入れ条件:
  - `runs/<profile_id>/<session_id>/` が生成される
  - `stdout.log` `commands.log` `meta.json` が保存される

### REQ-S1-006 思考モード選択
- 変更理由: グローバル設定を変更せずに、起動ごとに思考モードを切り替えるため
- 受け入れ条件:
  - UIで `low` `medium` `high` `xhigh` を選択できる
  - `Start Codex` 時に選択値が `-c model_reasoning_effort=<選択値>` として起動コマンドへ反映される

### REQ-S1-007 表示サニタイズ
- 変更理由: TTY制御シーケンスが可視テキストに漏れてノイズ表示（例: `MMM`）になる問題を防ぐため
- 受け入れ条件:
  - `ESC M` を含む出力がGUI上で `M` として残らない
  - 既存の `CSI` / `OSC` 除去挙動を維持する

### REQ-S1-008 Codex先頭プロンプト強調
- 変更理由: Codexの返答開始位置を視認しやすくするため
- 受け入れ条件:
  - Codex出力行の先頭 `›` / `>` 記号だけを薄い赤で表示する
  - 本文テキスト色（`out`）は維持する

### REQ-S1-009 送信方式の根本切替
- 変更理由: Codex TUIで `text+Enter` 同時送信時に初回取りこぼしが起きるため、送信プロトコルを「文字列送信」と「Enter送信」に分離して確実化する
- 受け入れ条件:
  - `Send` ボタン押下で送信できる
  - 入力欄で `Enter` / テンキー `Enter` を押しても送信しない（使用不可）
  - 送信時は、TTYへ「文字列本体」を先に書き込み、短い反映待ち後に `Enter` を送る
  - 空文字または空白だけの入力は送信しない（`commands.log` に空の `[user]` 行を残さない）

### REQ-S1-010 対象プロジェクト表示
- 変更理由: どのプロジェクトに対してCodexを起動しているかを常時確認できるようにし、誤操作を減らすため
- 受け入れ条件:
  - メイン画面に `Project Name` と `Target Path` が表示される
  - `Project Name` は、要件定義書内 `プロジェクト名` -> 要件定義ファイル名 -> フォルダ名 の順で解決される
  - 親画面から `Project Details...` 子モーダルを開き、対象情報（Profile/Project Name/Target Path/Requirement File）を確認できる

### REQ-S1-011 Profile管理モーダル
- 変更理由: Profileを `profiles.json` 手編集なしで管理し、プロジェクト切替をGUIで完結させるため
- 受け入れ条件:
  - 親画面から `Manage Profiles...` 子モーダルを開ける
  - 子モーダルで `Add/Edit/Duplicate/Delete` ができ、`Apply` で `profiles.json` へ保存される
  - 保存後に親画面のProfile選択肢が再読込される
  - セッション実行中はProfile管理を無効化する

### REQ-S1-012 プロジェクト名不一致警告の廃止
- 変更理由: 運用上、フォルダ名と要件定義由来のプロジェクト名が一致しないケースが通常であり、不一致警告が常時ノイズになるため
- 受け入れ条件:
  - `Project Name` とフォルダ名の不一致では警告を表示しない
  - 要件定義ファイルが見つからない場合の警告は維持する

### REQ-S1-013 対話の別ウィンドウ化
- 変更理由: 別ウィンド運用の対象を設定系（プロファイル管理・詳細確認）に限定し、対話操作はメイン画面で完結させるため
- 受け入れ条件:
  - 対話表示/入力はメイン画面に表示される
  - `プロジェクト詳細` / `プロファイル管理` は子ウィンドウ（`Toplevel`）で開く

### REQ-S1-014 UI日本語化
- 変更理由: 日常運用の直感性を上げるため
- 受け入れ条件:
  - 親画面、設定モーダルの主要ラベル/ボタン/警告を日本語に統一する
  - セッション状態表示（停止中/実行中）を日本語で表示する

### REQ-S1-015 pygame実行基盤の導入（ウィンドウ生成除く）
- 変更理由: 指定方針に合わせ、`pygame` をUIの見た目設定に活用するため
- 受け入れ条件:
  - UI配色生成に `pygame` を利用する
  - `pygame` が使えない場合はフォールバック配色を使う

### REQ-S1-016 対象プロジェクト表示の背景統一
- 変更理由: `対象プロジェクト` パネル周辺に白背景が見えて視認上の違和感があるため
- 受け入れ条件:
  - `対象プロジェクト` パネルの内外背景をメイン背景と同系グレーで統一する
  - ラベルフレームとラベル文字が既存可読性を維持する

### REQ-S1-017 背景薄青の濃度調整
- 変更理由: 全体背景の薄青が薄すぎるため、視認上のコントラストを少し強めるため
- 受け入れ条件:
  - メイン背景色を現状より一段濃い薄青へ調整する
  - 対話ログ領域背景も同系色で一段濃く調整する
  - `pygame` 経路とフォールバック経路で同一カラー値を返す

### REQ-S1-018 Codex表示領域の白寄せ調整
- 変更理由: Codexテキスト表示部分の可読性をさらに上げるため
- 受け入れ条件:
  - Codex出力表示領域（テキストビュー背景）を白に近い色へ調整する
  - 全体背景の薄青トーンは維持する
  - `pygame` 経路とフォールバック経路で同一カラー値を返す

### REQ-S1-020 プロファイル既定思考レベル
- 変更理由: プロジェクトごとに既定思考レベルを保持し、プロファイル切替時の再設定手間を減らすため
- 受け入れ条件:
  - プロファイル管理モーダルの追加/編集で `既定思考レベル`（`low` `medium` `high` `xhigh`）を設定できる
  - 設定値が `profiles.json` に保存され、再読込後も維持される
  - プロファイル選択時に、親画面の思考レベルコンボが該当プロファイル既定値へ同期される

### REQ-S1-021 表示テーマ表示の削除
- 変更理由: 親画面の情報密度を下げ、不要な状態表示を減らすため
- 受け入れ条件:
  - 親画面から `表示テーマ` の文言を削除する
  - 既存の `Codex` 可用状態表示と `状態` 表示は維持する

### REQ-S1-022 VoiceAttack連携向け単一起動・外部操作基盤
- 変更理由: 外部音声入力ソフト（VoiceAttack）から誤対象への入力を防ぎ、既存ウィンドウへ確実に操作を集約するため
- 受け入れ条件:
  - アプリは多重起動を禁止し、2重起動時は新規起動せず既存ウィンドウ側へ操作フォワードして終了する
  - 外部操作で入力欄フォーカス制御（`--focus-input`）ができる
  - 外部操作（前面化/入力欄フォーカス）の監査ログを `runs/external_audit.log` へ記録する

### REQ-S1-023 ウィンドウタイトル固定化
- 変更理由: VoiceAttackのウィンドウ選択を完全固定化し、状態変化時のタイトル不一致をなくすため
- 受け入れ条件:
  - ウィンドウタイトルは常に同一文字列（`Codexラッパー`）で表示する
  - プロジェクト/プロファイル/状態の変更によってタイトル文字列が変化しない

### REQ-S1-024 実行中背景の薄緑切替
- 変更理由: 実行状態を視覚的に即判別できるようにするため
- 受け入れ条件:
  - 停止中は既存の薄青背景を維持する
  - 実行中のみ、メイン背景とパネル背景を薄い緑系へ切り替える
  - セッション停止時は背景を薄青に戻す

### REQ-S1-025 入力欄フォーカスボタンの廃止
- 変更理由: メイン画面のボタン密度を下げ、必要最小限の常設ボタンに整理するため
- 受け入れ条件:
  - メイン画面に `入力欄フォーカス` ボタンを表示しない
  - 外部操作による入力欄フォーカス機能は維持する

### REQ-S1-026 exe実行時の外部プロセス起動互換
- 変更理由: PyInstaller化したexeから `pwsh.exe` 起動時に `0xc0000142` が発生するケースを回避し、Codex起動を安定化するため
- 受け入れ条件:
  - 外部プロセス起動時、frozen環境由来のDLL検索パス競合を避ける実行経路を使う
  - frozen時のPATHから `_MEIPASS` 相当パスを除外した環境を子プロセスへ渡せる
  - 非frozen実行では既存挙動を維持する

### REQ-S1-027 exe既定保存先と既定作業フォルダの安定化
- 変更理由: onefile exe実行時に `profiles/runs` が一時 `_MEI` 配下へ流れ、多重起動判定とプロファイル保存先が不安定になるため
- 受け入れ条件:
  - exe実行時の既定 `profiles-path` と `runs-root` は `_MEI` ではなく永続パスを解決する
  - `dist` 配置のexeでは既定保存先を `dist` 親ディレクトリへ寄せる
  - 初回生成 `default` プロファイルの `working_directory` は `C:\Users\gonec\codex_workspace`（存在時）を優先する
  - `_MEI` 配下を指す既存プロファイルは起動時に既定作業フォルダへ正規化される

### REQ-S1-028 pwsh初期化失敗時の自動フォールバック
- 変更理由: 環境依存で `pwsh` が `0xC0000142` で即終了する場合でも、ラッパー操作を継続可能にするため
- 受け入れ条件:
  - `pwsh` 起動直後に `0xC0000142` を検知した場合、`powershell.exe` へ自動切替してセッション開始を継続する
  - セッション開始ログに自動切替した事実を明示する
  - `meta.json` に要求シェルと実際に使用したシェル、フォールバック適用有無を記録する

### REQ-S1-029 exe配置自由化と自己初期化
- 変更理由: exeの配置先が可変でも起動できるようにし、必要ファイル/フォルダをアプリ自身で初期化するため
- 受け入れ条件:
  - 既定保存先（`profiles.json` / `runs`）は実行環境で書き込み可能なパスへ解決される
  - 配置先が書き込み不可の場合はユーザーデータ領域へ自動フォールバックする
  - `profiles.json` 親ディレクトリと `runs` ディレクトリは起動時に自動生成される
  - frozen実行で `pwsh` 指定時は互換目的で `powershell.exe` を優先使用できる

### REQ-S1-030 入力送信失敗時の再送容易化
- 変更理由: Enter送信時に送信されないケースで入力文字列が消えると再送操作が重くなるため
- 受け入れ条件:
  - 送信失敗時（セッション未実行/例外）は入力欄文字列を保持する
  - Enter送信は `after_idle` で実行し、キー入力確定後の文字列を送信対象にする
  - 送信成功時のみ入力欄をクリアする
